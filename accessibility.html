Den automatiske testen

Den automatiske testen gir kun to feil, som beror på at <a> elementene som inneholder logo og meny-burger ikke har noen tekst, men siden disse inneholder visuelle elementer ignorerer jeg disse feilvarslingene.

Testen oppgir også at siden inneholder språkdefinering <html lang="nb">, samt aria-hidden="true" på ikoner for å hjelpe skjermlesere. Alternativ tekst er også oppgitt på bilder som ikke er overskriftsbilder, men som er del av innholdet, og disse bildene er også hentet med html i stedet for css.

Siden er strukturert etter hovedelementene <header>, <main> og <footer>, deretter inndelt etter med <nav>, <section> og hierarkiske headers.

CSS

Hva gjelder CSS-en så er den bygget opp «mobile first», og deretter skaleres den opp med @media query.

@media (min-width: 800px) {
}

Det er også bevisst valgt at noen av selectorene er relative slik at om man ville byttet en seksjon fra lys til mørk så endres også fargene på <p>, <h#> og <a> også automatisk. Andre elementer som knapper og kode, som man muligens vil ha mer styring på selv er designet med mulighet for å selv velge når man vil bruke mørk og lys variant, eksempelvis:

/*Relativ selector*/
.section-light a {
color: #0C1821;
}
/*Manuelt fargevalg*/
.code-blue-light {
color: deepskyblue;
}
.code-blue-dark {
color: midnightblue;
}
